01
设计理念与原则

1.1 设计理念

云笔记的设计理念为 "轻量、专注、高效"。我们追求让用户在最短时间内进入写作状态，最大限度减少界面干扰，让内容本身成为视觉焦点。

理念 释义 设计体现
轻量 界面简洁不臃肿，减少视觉噪声 极简图标、低饱和度配色、细线边框、大量留白
专注 让用户聚焦于内容创作，而非工具本身 三栏渐进式展开、工具栏悬停显示、编辑区最大化
高效 减少操作步骤，提供快捷路径 全局搜索、快捷键体系、右键菜单、拖拽操作

1.2 设计原则

原则 说明
一致性 相同类型的元素使用统一的视觉样式和交互方式。按钮、输入框、卡片等组件在全应用保持一致的圆角、间距和颜色
层次分明 通过颜色深浅、字号大小、字重变化构建清晰的视觉层级。最重要的信息最突出，次要信息逐级弱化
即时反馈 每个操作都有明确的视觉反馈。悬停高亮、点击状态、保存提示、Toast 通知等确保用户始终了解系统状态
容错设计 关键操作提供撤销能力（回收站、历史版本）。删除操作需要确认，防止误操作造成数据丢失
渐进展示 不一次性展示所有功能。工具栏按分组折叠，高级功能通过菜单或模态框触达，降低认知负担

09
主题系统

9.1 实现方案

通过 HTML 根元素 data-theme 属性 + CSS 自定义属性（CSS Custom Properties）实现主题切换，无需加载额外样式文件。

/* 亮色主题（默认） */ :root { --bg-primary: #ffffff; --text-primary: #1a1a2e; --accent: #4f6ef7; } /* 暗色主题 */ [data-theme="dark"] { --bg-primary: #1a1b2e; --text-primary: #e8e8f0; --accent: #6380ff; } /* 组件引用变量，无需关心当前主题 */ .sidebar { background: var(--bg-sidebar); color: var(--text-primary); }

12.1 过渡时长

场景时长缓动函数用途

12.2 动画效果

效果名称描述用途
fadeIn从 opacity:0 + translateY(-4px) 到 opacity:1 + translateY(0)，0.2s ease待办事项新增时的入场动画
hover lift悬停时背景色变化 + 可选微上移笔记卡片、导航项悬停
focus ring输入框聚焦时 3px 主色外发光所有输入框聚焦反馈
slide collapse宽度从 260px 到 0 的过渡侧边栏折叠

12.3 动效原则

⚠ 克制使用动效：动效服务于功能反馈，不应成为视觉干扰。
• 高频操作（输入、滚动、点击）：过渡时长 ≤ 0.2s
• 低频操作（面板展开、模态框弹出）：过渡时长 0.2s-0.3s
• 不使用弹跳、旋转等夸张动效
• 全局禁用硬件加速，优先使用 transform/opacity 避免重排
